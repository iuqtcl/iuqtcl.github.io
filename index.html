1.1 disk_manager

磁盘管理器主要实现了从磁盘中读写文件，以及创建删除文件。管理内存与磁盘间的交互。

数据结构

disk_manager中记录了

  1. 打开文件的文件路径 path 和 fd 码的对应关系，使用 unordered_map 储存；
  2. 打开的文件中已分配的页面数量，通过fd索引。

函数实现

write_page()：使用 lseek 函数，设置 fd 文件中的偏移量为 page_no * num_bytes，即定位到第 page_no 页的页首。通过返回值判断偏移量设置是否成功。使用 write() 从磁盘中读数据到指定位置offset。read_page() 类似。

AllocatePage():将记录中fd对应的已分配页面数量加1。

is_file()：使用 struct stat 获取文件信息。

创建和删除指定路径文件：分别使用了 open 和 unlink 函数。注意 string 类型的变量作为c语言下函数的参数时，需要使用 c_str() 函数转换为标准c字符串。

打开关闭指定路径文件：分别调用 open 和 close 函数。打开时添加 fd 和 path 的对应关系记录。关闭时移除。

1.2 lru_replacer

实现了最近最少使用策略的缓冲页面管理。当缓冲池需要替换页面时选择合适的页面。

数据结构

lru_replacer中记录了

  1. latch_ 信号量类型作为互斥锁。使用时采用了 C++17 std::scoped_lock 智能锁，调用构造函数时自动上锁，析构函数自动解锁。
  2. 按加入时间顺序排列的list。元素类型为 frame_id_t。首部表示最近被访问。
  3. LRUhash_ 快速查找 list 中指定的 frame_id，便于修改列表。

特殊数据类型：

frame_id_t 内存中的frame_id类型。用于表示内存中的空间。frame大小与page一致。

函数实现

Victim 寻找需要淘汰的页面。将LRUlist末尾的元素取出，从 LRUhash 和 LRUlist 中移除页面。

Pin/Unpin 固定/取消固定页面。从 LRUhash 和 LRUlist 中移除/添加页面。Unpin 中检查缓冲池内页面数量是否大于最大值，依次从 list 尾部删去过多的页面。

Size 可以 victim 的数量。返回LRUlist中的元素数量即可。

1.2 语义、解释器和栈式虚拟机

堆、栈 内存划分

栈内存

每个函数被调用时，创建一个对应本次函数调用的 stack frame 并 push 到栈内存上，当这个函数返回时 pop。

每个栈帧的大小等于当前函数所有的形参和局部变量大小的总和加元数据。变量名在编译时转换为偏移量。

多线程程序，每个线程都有自己的栈内存。

优势:

  1. 无需通过指针或引用动态寻址，访问速度快。
  2. 生命周期明确，内存空间使用效率高。
  3. 大小和偏移量固定。
  4. 不需要复杂的垃圾回收机制。

特性：

  1. 函数不能返回一个指向栈内存的结构体的指针。因为返回后数据就被删除了。（但对于C/C++，形参或局部变量是一个指针，变量大小只考虑指针本身的大小，不考虑指针指向的数据的大小）
  2. 无法在多个线程间共享。
  3. 大小较小（几MB），函数调用过深或结构体过大会 stack overflow

堆内存

和数据结构中的堆没有关系。

按需分配的动态内存管理机制。申请时再分配（相对于栈内存调用函数时一次性分配）。因此访问时需要通过指针或引用。

常见堆内存管理手段

  1. 程序员手动管理（C/C++)
  2. 垃圾回收机制
  3. 程序员明确指出每个变量的声明周期，并引入结构体的转让/借用机制。（Rust）

什么是(高级语言)虚拟机（HLL VM)

用于实现高级语言的语义。（主流HLL VM，诸如JVM、CLI、CPython、CRuby 1.9）

输入： 满足某种指令集架构的指令序列 （可能由编译器生成）

处理： 转换为目标ISA的指令序列并加以执行（编译/解释实现）

输出： 程序的执行结果

Smart Pointers

put SMFs to good use

在constructor中acquire，在destructor中release

  • std::unique_ptr
    • Uniquely owns its resource, can’t be copied
  • std::shared_ptr
    • Can make copies, destructed when underlying memory goes out of scope
  • std::weak_ptr
    • Models temporary ownership: when an object only needs to be accessed if it exists (convert to shared_ptr to access)
